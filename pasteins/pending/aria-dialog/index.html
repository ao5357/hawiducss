The CSS framework you should be using.

WAI-ARIA Dialog

Modal dialogs that can require focus or user action, per the ARIA spec. To use, paste the following CSS into styles.css:

[role=alertdialog],[role=dialog]{background:#fff;border:1px solid #999;border-radius:.5em;box-shadow:0 0 1em 100em rgba(0,0,0,.5);left:25%;padding:1em;position:fixed;top:15%;width:50%;z-index:999;}

Then, wherever you want a dialog, insert the following HTML where it can roam free in the body of the document (via fixed positioning). It doesn't have to be a div, just a block-display element that makes sense. You could do this programmatically, but be sure to test:

<div role="dialog"> This dialog can contain arbitrary HTML. <p>In container nodes or otherwise. Those buttons are for example only.</p> <button type="button">Accept</button>> <button type="button">Reject</button> </div>
Dialogs can contain arbitrary HTML. This dialog is positioned so that the page remains useful. Pastein code is centered.
